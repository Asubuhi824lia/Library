Разработка тестов для проверки корректности входных данных
1) при регистрации
2) при указании фильтров при поиске книги
3) при входе в аккаунт (допустимость введённых данных + соответствие предоставленного после функционала типу пользователя)
4) при приёме/выдаче книг

	Случай 1: регистрация
Критерии корректности полей:
1. ФИО
Должны содержать по 1 слову длиной не более 25 символов на русском или английском языке, исключая иные символы. Не может содержать меньше 2 символов.
2. Домашний адрес: без тестов.
3. Номер телефона
Поле должно содержать только цифры. Для фильтра "Мобильный" - 11 цифр, "Домашний" - 7

Критерии остальных полей повторяют Случай 4.

	Случай 2: фильтры
Критерии корректности полей:
1. Название
Состоит из любой строки до 100 символов. Может содержать любые символы.
2. Автор
Состоит из 3-х слов русского алфавита, может включать символ "-". Кол-во символов в строке не превышает 100. Допустимый формат: "Янов Ян Янович". Однако расположение Ф И О не важно.

	Случай 3: при входе в аккаунт
Критерии корректности полей:
1. Логин иметь длину от 3 до 20 символов и не должен содержать следующие символы: " / \ [ ] : ; | = , + * ? < > ` ’ { } @ # $ % ^ & ! № . Также нельзя использовать пробелы. Допустимые символы: буквы латинского алфавита (a–z), русского (а-я) и цифры (0–9). 
2. Длина пароля не должна превышать 16 символов. Допустимы те же символы, что и для логина, только вместе со специальными символами.

	Случай 4: при приёме/выдаче книг
Критерии корректности полей:
Пункты 1 и 2 повторяют случай 2.
3. Номер читательского билета
Состоит ровно из 13 цифр.
